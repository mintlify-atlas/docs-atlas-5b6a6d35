# Quick Start

Get up and running with base-react-v2 in just a few steps. This guide will have your development server running in under 5 minutes.

<Note>
This boilerplate is optimized for Bun but works with npm, yarn, or pnpm as well.
</Note>

## Prerequisites

Before you begin, ensure you have one of the following installed:

- [Bun](https://bun.sh) 1.0 or higher (recommended)
- [Node.js](https://nodejs.org) 18+ with npm, yarn, or pnpm

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the boilerplate to your local machine:

    ```bash
    git clone <repository-url>
    cd base-react-v2
    ```
  </Step>

  <Step title="Install dependencies">
    Install project dependencies with your preferred package manager:

    <CodeGroup>
      ```bash bun
      bun install
      ```

      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```

      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>

    <Tip>
    Using Bun? Installation is typically 3-5x faster than npm.
    </Tip>
  </Step>

  <Step title="Configure environment">
    Create your environment file from the example:

    ```bash
    cp .env.example .env.dev
    ```

    The default configuration includes:

    ```bash
    VITE_PUBLIC_URL=/
    VITE_TANKSTACK_ROUTER_TOOLS=true
    ```

    <Note>
    - `VITE_PUBLIC_URL`: Set the base path for your app (use `/` for root or `/subpath` for subdirectories)
    - `VITE_TANKSTACK_ROUTER_TOOLS`: Enable/disable TanStack Router devtools
    </Note>
  </Step>

  <Step title="Start the dev server">
    Launch the development server:

    <CodeGroup>
      ```bash bun
      bun run dev --mode dev
      ```

      ```bash npm
      npm run dev -- --mode dev
      ```

      ```bash yarn
      yarn dev --mode dev
      ```

      ```bash pnpm
      pnpm dev --mode dev
      ```
    </CodeGroup>

    Your app will be available at `http://localhost:5173` (Vite's default port).

    <Tip>
    The dev server runs on `0.0.0.0` by default, making it accessible from other devices on your network.
    </Tip>
  </Step>
</Steps>

## What You Get

Once the dev server is running, you'll have access to:

### Application Routes

The boilerplate comes with pre-configured feature modules:

- **Landing Pages** (`/`): Public-facing pages
- **Authentication** (`/auth/login`, `/auth/register`): Login and registration flows
- **Backoffice** (`/backoffice/*`): Protected admin dashboard
- **404 Handling**: Automatic redirect for unknown routes

## Quick Architecture Overview

The boilerplate uses a feature-based architecture:

```typescript
// src/App.tsx - Entry point
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import { Providers } from './core/providers/providers'
import { TankStackRouter } from './core/router/tankStackRouter'
import './app.css'

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <Providers>
      <TankStackRouter />
    </Providers>
  </StrictMode>,
)
```

<Tip>
The `Providers` component wraps your app with global providers (QueryClient, Theme, etc.). The `TankStackRouter` component handles all routing logic.
</Tip>

## Routing Example

Routes are defined per feature using factory functions:

```typescript
// core/router/tankStackRouter.tsx
const routeTree = rootRoute.addChildren([
  ...baseRedirectRouter(baseRoute, rootRoute),
  landingRouter(rootRoute),
  authRouter(rootRoute),
  backofficeRouter(rootRoute),
])

const router = createRouter({ routeTree })
```

Each feature module exports a router factory that creates its route subtree. This keeps features isolated and composable.

## Need Help?

If you run into issues:

1. Check that your Node.js/Bun version meets the requirements
2. Delete `node_modules` and reinstall dependencies
3. Verify your `.env.dev` file exists and has the correct variables
4. Check the browser console for error messages

<Tip>
Most issues can be resolved by ensuring environment variables are properly configured and dependencies are correctly installed.
</Tip>
