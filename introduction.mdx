### Key Features

#### Layered Architecture

Clean separation of concerns with a modular feature-based structure:

- **Core Layer**: Infrastructure and base configuration
- **Shared Layer**: Reusable components, hooks, and utilities
- **Features Layer**: Self-contained business logic modules

<Tip>
Each feature is completely independent with its own domain, infrastructure, and presentation layers. This makes your codebase highly maintainable and testable.
</Tip>

#### Feature-Based Organization

Organize code by business domain, not by technical role:

```bash
features/
├── auth/          # Authentication flows
├── landing/       # Public pages
├── backoffice/    # Admin dashboard
└── error404/      # Error handling
```

#### Built-in Best Practices

- **Singleton HTTP Client**: Centralized API configuration with interceptors
- **Adapter Pattern**: Clean API response transformation
- **Builder Pattern**: Composable router configuration
- **Route Guards**: Protect routes with reusable guard functions
- **Type-Safe Routing**: Full TypeScript support for routes and params

<Note>
The architecture follows SOLID principles and emphasizes dependency inversion - layers always depend inward toward the domain, never outward.
</Note>

### Bun Support

While this boilerplate works with npm, yarn, and pnpm, it's optimized for [Bun](https://bun.sh) - the all-in-one JavaScript runtime that's significantly faster than Node.js:

```bash
bun install    # ~3x faster than npm install
bun run dev    # Instant dev server startup
bun run build  # Optimized production builds
```

## Project Structure

The boilerplate follows a clean architecture with three main layers:

```bash
src/
├── core/                    # Infrastructure base
│   ├── providers/           # Global providers (QueryClient, Theme)
│   └── router/              # TanStack Router singleton
│
├── shared/                  # Reusable across features
│   ├── components/ui/       # Generic UI components
│   ├── constants/           # Global constants and routes
│   ├── guard/               # Route guards
│   ├── hooks/               # Shared hooks
│   └── utils/               # Pure utility functions
│
└── features/                # Business logic modules
    ├── auth/
    ├── landing/
    ├── backoffice/
    └── error404/
```

<Tip>
Features never import from other features. All shared code lives in `shared/`. This prevents tight coupling and makes code easier to maintain.
</Tip>

## Feature Anatomy

Each feature follows a clean architecture pattern with clear separation:

```bash
features/example/
├── domain/              # Types and interfaces
├── infrastructure/      # API, cache, and state
│   ├── api/            # HTTP calls
│   ├── stacks/         # TanStack Query cache
│   └── store/          # Global state (Zustand)
├── application/         # Service orchestration
├── layout/              # Feature-specific layouts
├── pages/               # Route components
└── router/              # Route definitions
```

## Router Architecture

TanStack Router provides type-safe, file-based routing with powerful features:

- **Type-Safe Navigation**: Full TypeScript support for routes and params
- **Data Loaders**: Pre-fetch data before rendering routes
- **Route Guards**: Protect routes with authentication checks
- **Code Splitting**: Automatic lazy loading of route components
- **Devtools**: Built-in devtools for debugging routing

<Note>
The router is configured as a singleton in `core/router/tankStackRouter.tsx`. Each feature exposes a factory function that builds its route subtree, keeping features isolated.
</Note>
